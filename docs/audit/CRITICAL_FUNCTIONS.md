# Critical Functions

All functions in `cadence/contracts/PrizeLinkedAccounts.cdc`.

| Function | Access | Preconditions | What Could Go Wrong |
|----------|--------|---------------|---------------------|
| `deposit` | `PositionOps` via `PoolPositionCollection` | `balance > 0`; vault type matches pool asset; `totalAssets + amount <= SAFE_MAX_TVL`; `maxSlippageBps <= 10000`; state is Normal (enforce minimumDeposit) or PartialMode (enforce partialModeDepositLimit); panics if Emergency or Paused | Yield source silently swallows tokens (mitigated: `depositToYieldSourceFull` asserts `actualReceived > 0`). Slippage check uses nominal vs actual -- a malicious yield source returning inflated `minimumAvailable()` after deposit could bypass protection. |
| `withdraw` | `PositionOps` via `PoolPositionCollection` | `amount > 0`; receiver registered or sponsor; state not Paused | Returns empty vault on liquidity failure (no revert). Consecutive failures auto-trigger EmergencyMode. Dust threshold (`minimumDeposit / 10`) determines full-withdrawal coercion -- a very low minimumDeposit leaves dust. Unregistration blocked during active draw (`pendingSelectionData != nil`) to prevent swap-and-pop index corruption; ghost entries accumulate until cleanup. |
| `startDraw` | `CriticalOps` (Admin) or permissionless via contract-level `startDraw(poolID)` | State == Normal; `pendingDrawReceipt == nil`; `activeRound != nil`; `canDrawNow()` (round target time reached); health check passes | Asserts `allocatedPrizeYield > 0` -- if no yield accrued the draw reverts. Protocol fee recipient capability could be stale (invalid cap silently routes to unclaimed vault). Force-unwrap on `(&self.activeRound as &Round?)!` is guarded by precondition `activeRound != nil`. |
| `processDrawBatch` | `CriticalOps` (Admin) or permissionless via contract-level `processDrawBatch(poolID, limit)` | `limit >= 0`; `activeRound != nil`; `pendingDrawReceipt != nil`; `pendingSelectionData != nil`; batch not already complete | Snapshot count freezes receiver list length at `startDraw()`. New deposits during batch don't extend it (prevents DoS). Weight overflow asserted against `WEIGHT_WARNING_THRESHOLD` per `addEntry()`. Force-unwraps on selectionData and activeRound refs are guarded by preconditions. |
| `requestDrawRandomness` | N/A -- randomness is requested inside `startDraw()` via `randomConsumer.requestRandomness()` | Called internally by `startDraw` | Flow's `RandomConsumer` requires fulfillment in a different block. If `completeDraw` is called in the same block as `startDraw`, `fulfillRandomRequest` panics. |
| `completeDraw` | `CriticalOps` (Admin) or permissionless via contract-level `completeDraw(poolID)` | `pendingDrawReceipt != nil`; `pendingSelectionData != nil`; `isBatchComplete()` | PercentageSplit rounding could cause `calculatedSum >= totalPrizeAmount` (fixed in PR #55 -- last winner gets 0.0). PRNG seed was `s0 == s1` (fixed in PR #53). Prize amount asserted `<= allocatedPrizeYield` per winner. If no eligible participants, prizes carry forward (no distribution). |
| `syncWithYieldSource` | `access(contract)` -- called by deposit, withdraw, startDraw, and admin `processPoolRewards` | None (skips if difference < `MINIMUM_DISTRIBUTION_THRESHOLD`) | Reads `yieldConnector.minimumAvailable()` which is external -- a manipulated yield source can fabricate excess or deficit. Deficit waterfall: protocol fee -> prize -> rewards (share price). Insolvency event emitted but not enforced (no revert). |
| `enableEmergencyMode` | `CriticalOps` | `reason.length > 0` | Blocks deposits and draws but allows withdrawals. No automatic reversion unless `autoRecoveryEnabled`. Admin can lock pool indefinitely. |
| `disableEmergencyMode` | `CriticalOps` | None beyond pool existence | Resets `consecutiveWithdrawFailures` to 0 and returns to Normal. Can be called even if underlying yield source issue persists -- re-enables deposits into a broken source. |
| `setProtocolFeeRecipient` | `OwnerOnly` | Capability must pass `.check()` if provided | Redirects all future protocol fee. If recipient capability becomes invalid after setting, fee silently goes to unclaimed vault. Vault type compatibility validated via `isSupportedVaultType`. |
| `withdrawUnclaimedProtocolFee` | `CriticalOps` | `amount > 0` | Borrows recipient capability with `?? panic()` (fixed TOCTOU in PR #56 -- was `check()` then `borrow()!`). Capped at available unclaimed balance. |
| `createPool` | `CriticalOps` | None beyond valid `PoolConfig` and optional `EmergencyConfig` | No upper bound on pool count (`nextPoolID` increments). Yield connector is stored by reference -- a malicious connector could steal deposited funds. |
| `depositNFTPrize` | `ConfigOps` | Pool exists | Uses `<-!` (force-move) to prevent duplicate NFT IDs. NFT remains in contract storage until awarded or admin-withdrawn. |
| `claimNFTPrize` | `PositionOps` via `PoolPositionCollection` | Registered with pool; pending NFTs exist for receiver; valid `nftIndex` | Index-based claim -- if multiple pending NFTs, indices shift after removal. Off-by-one if user caches old index. |
| `withdrawNFTPrize` | `ConfigOps` | NFT exists in `nftPrizes` map | Admin can withdraw NFTs referenced by an active `PrizeDistribution`. If NFT is withdrawn during draw, `completeDraw` skips it gracefully (`borrowNFTPrize` nil check). |
| `updatePoolDistributionStrategy` | `CriticalOps` | Pool exists | Takes effect immediately for next `syncWithYieldSource`. Mid-round change alters yield split without retroactive adjustment. No validation that strategy percentages sum correctly beyond `FixedPercentageStrategy.init` precondition. |
| `updatePoolPrizeDistribution` | `CriticalOps` | Pool exists | Takes effect on next `completeDraw`. Changing winner count mid-draw (after `startDraw`) affects prize distribution. No lock during active draw phase. |

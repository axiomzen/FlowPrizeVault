# Invariants

All references to `cadence/contracts/PrizeLinkedAccounts.cdc`.

| # | Invariant | Where Enforced | How To Break It |
|---|-----------|----------------|-----------------|
| 1 | `userPoolBalance + allocatedPrizeYield + allocatedProtocolFee == yieldSourceBalance` (after sync) | `syncWithYieldSource()` reconciles on every deposit/withdraw/draw. Excess distributes via strategy; deficit uses waterfall (protocol -> prize -> rewards). `getTotalAllocatedFunds()` returns the sum. | Call a pool operation that skips sync (none exist today -- all mutating paths call `syncWithYieldSource()` when `needsSync()` is true). Alternatively, the yield source's `minimumAvailable()` returns a stale or manipulated value, causing the invariant to drift until next sync. |
| 2 | `normalizedWeight <= shares` (TWAB safety cap) | `finalizeTWAB()` line ~1467: `if normalizedWeight > shares { return shares }`. Also in `getCurrentTWAB()` line ~1504. | Set `targetEndTime` to before current block time via `setTargetEndTime()` -- guarded by assertion `newTarget >= now` when shortening. If `actualDuration == 0.0`, returns 0. The cap prevents weight amplification from rounding or float drift. |
| 3 | `sharePrice = (totalAssets + VIRTUAL_ASSETS) / (totalShares + VIRTUAL_SHARES)` with `VIRTUAL_* = 0.0001` | `ShareTracker.getSharePrice()` line ~1778. `convertToShares` and `convertToAssets` delegate to this. | Division by zero impossible since `VIRTUAL_SHARES = 0.0001` ensures denominator >= 0.0001. An attacker donating assets without minting shares (e.g., sending tokens directly to yield source) inflates `totalAssets` at next sync -- virtual offset limits impact to ~0.0001% dilution for existing holders. |
| 4 | `sum(userShares[*]) == totalShares` | `ShareTracker.deposit()` increments both user and total. `ShareTracker.withdraw()` decrements both. No other mutation paths exist for these variables. | Bug in share arithmetic. `withdraw()` has dust prevention: burns all shares if `remainingValue < dustThreshold`, which could create a mismatch if `dustThreshold` is set incorrectly. Today `dustThreshold = minimumDeposit / 10` -- safe as long as `minimumDeposit >= 0`. |
| 5 | `withdrawal <= yieldSource.minimumAvailable()` | `Pool.withdraw()` checks `yieldAvailable < withdrawAmount` before calling `withdrawAvailable`. Returns empty vault on failure. For full withdrawals, caps at `yieldAvailable`. | Yield source's `minimumAvailable()` returns a value higher than what `withdrawAvailable()` actually provides. The code handles this gracefully: if `actualWithdrawn == 0.0` after the call, it emits `WithdrawalFailure` and returns empty vault. |
| 6 | No unregistration during active draw batch | `Pool.withdraw()` line ~3855: `if newShares == 0.0 && self.pendingSelectionData == nil` -- only unregisters when no batch in progress. `cleanupStaleEntries()` precondition: `self.pendingSelectionData == nil`. | Call `cleanupPoolStaleEntries` while `pendingSelectionData` exists -- blocked by precondition. Withdraw to 0 during batch -- user becomes ghost (0-weight, skipped in draw). Unregistered after draw via cleanup. |
| 7 | Valid state machine transitions: Normal <-> Paused, Normal <-> EmergencyMode, Normal <-> PartialMode | `setState()` accepts any `PoolEmergencyState`. `setEmergencyMode()`, `setPartialMode()`, `clearEmergencyMode()` are direct setters. `checkAndAutoTriggerEmergency()` only triggers from Normal. `checkAndAutoRecover()` only recovers from EmergencyMode. | `setPoolState()` (admin) can transition between any states without restriction (e.g., Paused -> EmergencyMode). This is by design for admin override, but a compromised admin could oscillate states. No guard prevents Emergency -> Paused bypassing recovery logic. |
| 8 | `actualEndTime >= targetEndTime` (round timing) | `startDraw()` sets `actualEndTime = getCurrentBlock().timestamp` only after `canDrawNow()` confirms `timestamp >= targetEndTime`. `setTargetEndTime()` asserts `newTarget >= now` when shortening. | Admin extends `targetEndTime` after `startDraw()` -- blocked by precondition `actualEndTime == nil` in `setTargetEndTime()`. The only setter for `actualEndTime` is `startDraw()`, which runs after `hasEnded()` check. |
| 9 | `allocatedPrizeYield >= 0` after prize distribution | `completeDraw()` asserts `prizeAmount <= self.allocatedPrizeYield` per winner. PercentageSplit rounding overflow guard (PR #55) prevents last-winner underflow. | Bug in a custom `PrizeDistribution` implementation that returns amounts summing to more than `totalPrizeAmount`. Each winner's amount is individually asserted, so this would panic on the exceeding winner. |
| 10 | Receiver list index consistency: `registeredReceivers[id] == index` and `registeredReceiverList[index] == id` | `registerReceiver()` appends to list and stores index. `unregisterReceiver()` uses swap-and-pop and updates moved element's index. | Concurrent modification during iteration. Protected by: batch uses `snapshotReceiverCount`, unregistration blocked during draw, cleanup blocked during draw. |
